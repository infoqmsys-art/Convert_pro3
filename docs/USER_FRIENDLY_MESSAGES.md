# 사용자 친화적 메시지 개선 가이드

## 개선 원칙

1. **"경고" → "안내"**: 더 부드러운 표현 사용
2. **"오류" → 구체적 설명**: 무엇이 잘못되었는지 명확히 안내
3. **기술적 용어 제거**: 일반 사용자가 이해하기 쉬운 표현
4. **친절한 톤**: "~하세요" → "~해주세요", "~합니까?" → "~하시겠습니까?"

## 주요 개선 사항

### 1. context_menu.py

**Before:**
- "경고", "업로드할 폴더를 선택하세요."
- "오류", "폴더 경로가 존재하지 않습니다."
- "재귀 업로드", "하위 항목도 함께 업로드합니까?"

**After:**
- "안내", "업로드할 폴더를 먼저 선택해주세요."
- "오류", "선택한 폴더를 찾을 수 없습니다.\n폴더 경로를 확인해주세요."
- "업로드 옵션", "하위 폴더의 파일들도 함께 업로드하시겠습니까?"

### 2. company_management_ui.py

**Before:**
- "경고", "'{company_name}' 업체가 이미 존재합니다."
- "완료", "업체 '{company_name}'가 추가되었습니다."
- "삭제 확인", "업체 '{company}'를 삭제하시겠습니까?\n\n업체 내의 모든 현장, 폴더, 파일 설정이 삭제됩니다."

**After:**
- "안내", "'{company_name}' 업체가 이미 등록되어 있습니다."
- "추가 완료", "업체 '{company_name}'가 성공적으로 추가되었습니다."
- "삭제 확인", "업체 '{company}'를 삭제하시겠습니까?\n\n⚠ 주의: 이 업체의 모든 현장, 폴더, 파일 설정이 함께 삭제됩니다.\n이 작업은 되돌릴 수 없습니다."

### 3. unregistered_files_ui.py

**Before:**
- "경고", "등록할 파일을 선택하세요."
- "등록 완료", "{added_count}개의 파일이 등록되었습니다.\n트리뷰를 새로고침하세요."

**After:**
- "안내", "등록할 파일을 먼저 선택해주세요."
- "등록 완료", "{added_count}개의 파일이 성공적으로 등록되었습니다.\n\n등록된 파일은 트리뷰에서 확인할 수 있습니다."

### 4. main_ui.py

**Before:**
- "경고", "삭제할 항목을 선택하세요."
- "삭제 확인", "현장 '{site}'를 삭제하시겠습니까?"

**After:**
- "안내", "삭제할 항목을 먼저 선택해주세요."
- "삭제 확인", "현장 '{site}'를 삭제하시겠습니까?\n\n⚠ 주의: 이 현장의 모든 폴더와 파일 설정이 함께 삭제됩니다."

## 적용 방법

각 파일의 messagebox 호출 부분을 위의 개선 사항에 맞게 수정하면 됩니다.

## 추가 개선 제안

1. **도움말 툴팁 추가**: 복잡한 기능에 대한 설명 추가
2. **단계별 안내**: 복잡한 작업을 단계별로 안내
3. **성공/실패 피드백 강화**: 작업 결과를 더 명확하게 표시
4. **에러 복구 제안**: 오류 발생 시 해결 방법 제시
