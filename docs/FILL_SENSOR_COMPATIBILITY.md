# 누락보충(Fill Interval) × 센서 설정 호환성 가이드

## 0. 주기 vs 누락보충 (핵심 구분)

| 구분 | 주기 설정 | 누락 보충 |
|------|-----------|-----------|
| **조건** | 원본에 **새 데이터 0행** (아예 안 들어옴) | 원본에 **일부 데이터 들어옴** (가끔 하나씩) |
| **처리** | base_time ~ 현재 시각 구간을 **주기(interval)대로 가라 행 생성** | 들어온 실데이터 **사이 구간**을 이전 값 복사로 채움 |
| **출처** | 마지막 변환 행(last_row) 값 복사 | 실데이터 처리 후, 이전 행 값 복사 |
| **의미** | "측정 자체가 없음" → 완전 가라 | "실측 데이터가 있으나 구간이 비어 있음" → 그 구간 보충 |

**목표**: 시간당 정확히 1개 (interval 기준), 없어도 안 되고 더 있어도 안 됨. 로직적으로 깔끔하게 처리.

---

## 1. 누락보충의 의미

- **원본에 특정 시간 데이터가 없음** → 해당 시간을 **이전 시간 값으로 보충**
- 예: 16:00, 18:00만 있고 17:00이 없으면 → 17:00 행을 **16:00 데이터 복사**로 생성
- 이렇게 할 때 **센서별 특성**에 맞아야 함

---

## 2. 변환 파이프라인 순서 (핵심)

```
[원본] → 시간필터 → 배터리이동 → 센서처리 → 소수점 → 누락보충 → 저장
```

- **누락보충은 센서 처리 이후**에 수행됨
- 채워지는 행 = **이전 행 전체 복사** (timestamp만 변경)
- 즉, "그 시간에 측정 없음" = "값 변화 없음" = **이전 값 유지**가 원칙

---

## 3. 센서 유형별 호환성

| 유형 | 대표 모드 | 특성 | 누락보충 적합 여부 |
|------|----------|------|-------------------|
| **행별 독립** | EL, V, OFFSET, SET, ANSAN_WM, EL_TAEAM, CHANG_V, CHANG_SM, TS, NM, EL_LOW, EL_STATION, EL_TUNNEL, VIBROMETER | 각 행의 원본값에 base/scale 적용 | ✅ 이전 값 유지 = 적합 |
| **누적 이어가기** | FM | `__last_converted_row__`로 직전 변환본 값 이어감, 월~토 06~18시에만 증가 | ✅ 측정 없음 = 증가 없음 = 이전 값 유지 |
| **완전 생성** | CR, CR_TAEAM, BASE_RAND | 원본 무시, base + cumsum/랜덤으로 가라값 생성 | ✅ 채워진 행은 이전 행의 생성값 복사 = 적합 |
| **참조 복사** | COPY | 다른 컬럼 값을 그대로 복사 | ✅ 이전 행 복사 = 적합 |

### 요약

- **모든 센서 모드**에 대해 "이전 행 복사"가 의미적으로 올바름
- 누락 시간 = 측정 없음 → 값 변화 없음 → 이전 값 유지

---

## 4. FM 센서 특수 케이스

- FM은 **무조건 변환본 마지막 값** 기준으로만 이어감 (원본/BASE fallback 없음)
- 변환본에 데이터가 없으면 0.0 사용 → 사용자가 변환본에 수동 입력 후 사용
- 채워진 행은 **센서 처리 이후** 생성되므로, 복사되는 "이전 행"에 이미 FM 값이 들어 있음
- 해당 시간대에 측정이 없으면 FM도 **증가하지 않음** → 이전 값 유지가 정확함

---

## 5. 검증 포인트 (로직 체크리스트)

1. **파이프라인 순서**  
   - 누락보충이 **센서 처리 뒤**에 수행되는지 확인

2. **채워진 행의 출처**  
   - 이전 행 전체 복사 + timestamp만 변경인지 확인

3. **last_row 사용**  
   - 증분 변환 시 변환본 마지막 행을 앞에 붙여 구간 연결  
   - FM 등 `__last_converted_row__` 사용 센서와 정합성 유지

4. **deviceId/STX 등 메타**  
   - 이전 행이 이미 채워진 행(0,0)이면 **다음 행(실측)**에서 가져오도록 처리

5. **시간 정렬**  
   - 누락보충 결과가 항상 시간순으로 정렬되어 저장되는지 확인

---

## 6. 새 센서 모드 추가 시 확인 사항

- 누락보충 = **이전 행 전체 복사**라는 전제가 깨지면 안 됨
- **행 간 의존성**이 특수한 경우 (예: 이전 행이 아닌 다른 기준으로 계산)  
  → 누락보충 로직과 충돌 가능성 검토 필요
- `__last_converted_row__`를 쓰는 센서는, 변환본 마지막 행이 올바르게 전달되는지 확인
