# 계측관리 웹 포털 — 구조 안내 (코드 없이 읽는 버전)

이 문서는 **버튼·메뉴가 무엇과 연결되는지**, **업체 → 현장 → 로거 → 센서** 순서가 어떻게 이어지는지만 정리했습니다.  
개발자용 상세 매핑은 [SCREEN_MAP_AND_ENTITIES.md](./SCREEN_MAP_AND_ENTITIES.md), 데이터 모델은 [DATA_MODEL_PHASE2.md](./DATA_MODEL_PHASE2.md)를 참고하면 됩니다.

---

## 1. 한 줄로 요약

웹 프로그램은 **SQLite 데이터베이스** 안에 다음 순서로 정보가 붙습니다.

```
업체 → 현장 → 로거 → 센서 → (나중에 쌓이는) 측정값·그래프
```

- **업체**: 회사·조직 단위 (예: 시공사, 건설사).
- **현장**: 업체 산하 현장 하나 (한글 이름 + 영문 코드 등).
- **로거**: 현장에서 데이터를 모으는 장비·단말 한 대 단위 (이름은 보통 `현장코드_0`, `_1` …).
- **센서**: 그 로거에 달린 측점·채널 하나 (라벨, 단위, 파일에서 몇 번째 열인지 등).

---

## 2. 큰 그림 (어디서 무엇을 만드나)

```text
[로그인]
    ↓
[업체 리스트] ———————— 새 업체 만들기 (관리자)
    ↓ 업체 선택
[현장 리스트] ———————— 현장 추가 (영문 코드 등)
    ↓ 현장 카드 클릭
[현장 워크스페이스] ← 여기서 매일 보는 화면
    ├ 현장설정    … 현장 정보·SMS 등 (현장 단위)
    ├ 센서설정    … 로거 추가·센서 추가·열 번호 (로거·센서 단위)
    ├ 카테고리    … 측정물 분류 트리 (준비·안내 단계 포함 가능)
    └ 좌측 트리    … 등록된 센서 목록 → 클릭 시 우측 그래프
```

---

## 3. 주요 버튼과 “무엇에 붙나” 표

| 화면에서 보이는 것 | 대략 하는 일 | 데이터 구조에서 어디에 해당하나 |
|-------------------|-------------|--------------------------------|
| **업체 리스트** (`/org-list`) | 접근 가능한 업체 목록, 업체 선택 | **업체(organization)** |
| **현장 리스트** (`/site-list`) | 그 업체의 현장 목록, **현장 추가** | **현장(site)** (반드시 업체 소속) |
| **현장 워크스페이스** (`/site/<번호>/workspace`) | 현장 허브: 트리·그래프·아래 버튼들 | **현장** 중심으로 로거·센서 조회 |
| **현장설정** | 주소·프로그램명·SMS·담당자 등 | **현장(site)** 및 현장 부속 설정 |
| **센서설정** | 로거 목록, 로거 추가, 센서 추가(파일 열 번호 등) | **로거(logger_device)** · **센서(sensor_channel)** |
| **로거 경로 설정** (링크로 열리는 화면) | CSV가 있는 **폴더 경로**, 시간 열 번호 등 | **로거** 한 대 설정 |
| **센서 채널 (레거시)** (있을 경우 링크) | 채널 상세·고급 옵션 | **센서** 한 줄 |
| **계측기 리스트** (워크스페이스 좌측) | 카테고리·미배치 센서까지 트리 표시 | **센서**들을 그룹과 함께 보여 줌 → 클릭 시 시계열 |

**정리**:  
- **현장설정** = 현장 자체 정보.  
- **센서설정** = 그 현장 안의 **로거·센서를 등록·편집**하는 곳.  
- **워크스페이스 트리** = 등록이 끝난 센서를 **조회·그래프**로 보는 곳.

---

## 4. 왜 “로거”와 “센서”가 둘 다 있나

- **로거**: “파일이 들어 있는 폴더”, “FTP/manual 종류”, “시간이 CSV 몇 번째 열인지”처럼 **파일·통신 단위** 설정에 가깝습니다.
- **센서**: “이 로거 파일 안에서 **값이 몇 번째 열**인지”, 이름·단위·관리기준 등 **측점 하나** 단위입니다.

자동으로 CSV를 DB에 넣는 프로그램(자동 DB 업로드)도 **로거 단위로 파일을 읽고**, 각 행에서 **센서에 적어 둔 열 번호**로 값을 가져옵니다.  
즉 **로거 없이 센서만**, 또는 **폴더만 있고 센서 정의 없음**이면 업로드 단계에서 건너뛰거나 저장할 곳이 없을 수 있습니다.

---

## 5. Convert Pro(변환 프로그램)와의 관계

화면 왼쪽 안내 문구와 같이, 이 포털은 **전용 SQLite DB**를 씁니다.  
**Convert Pro(`Convert_pro3.py`)와 프로그램·DB를 공유하지 않습니다.**  
연동이 필요하면 CSV 업로드·경로 지정·운영 규칙으로 맞추는 방식입니다.

---

## 6. 계정과 보이는 범위 (아주 간단히)

- **최고 관리자**: 업체·현장 전반, 사용자 계정까지.
- **계측자(편집)**: 할당된 현장만, 설정 수정 가능한 경우가 많음.
- **게스트**: 할당된 현장 **조회 위주**.

실제 번호(`access_level`)는 시스템 관리 문서를 따릅니다.
